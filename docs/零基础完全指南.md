## 一、项目简介

WebMDAI 是一个命令行工具，可以帮你：

| 功能           | 作用               | 比喻         |
| ------------ | ---------------- | ---------- |
| **fetch**    | 爬取网页，保存为Markdown | 像爬虫一样下载网页  |
| **deal**     | 批量替换/删除文本        | 像Word的查找替换 |
| **llm**      | 调用AI处理内容         | 让AI帮你翻译/摘要 |
| **workflow** | 自动化多步骤           | 一键完成上面所有   |

**工作流程示例**：

```
网页链接 → [fetch爬取] → Markdown文件 → [deal清理] → 干净内容 → [llm翻译] → 中文结果 → [合并] → 成品
```

## 三、文件都存哪了？（超级重要）

很多新手搞不清楚文件都存在哪里，这是完整说明：

### 3.1 配置文件（只此一处）

```
位置：C:\Users\你的用户名\.webmdai\config.json

作用：存储模型配置、默认设置
```

**示例内容**：

```json
{
  "default_model": "deepseek",
  "models": {
    "deepseek": {
      "endpoint": "https://openrouter.ai/api/v1",
      "model": "deepseek/deepseek-chat",
      "key": "sk-or-v1-xxxx"
    }
  },
  "fetch": {
    "default_reader": "jina",
    "timeout": 30,
    "delay": 1.0
  }
}
```

### 3.2 工作目录（你在哪运行，就在哪生成）

```
假设你在 E:\MyNovel 目录运行命令
↓
所有输出文件都在 E:\MyNovel 下生成
```

### 3.3 爬取输出文件结构

运行 `webmdai fetch` 后会生成：

```
E:\MyNovel\
├── 任务名_1_标题1.md        ← 第1个网页
├── 任务名_2_标题2.md        ← 第2个网页
├── 任务名_3_标题3.md        ← 第3个网页
└── 任务名_合并.md           ← 所有网页合并（如果用了-st）
```

每个文件开头都有元数据：

```yaml
---
fetch_time: 2026-02-23T17:50:50  ← 爬取时间
source_url: https://example.com  ← 原始网址
title: 页面标题
---
```

### 3.4 LLM处理输出文件结构

运行 `webmdai llm` 后会生成：

```
E:\MyNovel\任务名\              ← 创建子目录
├── 任务名_1_标题1_translate.md  ← 第1个文件翻译结果
├── 任务名_2_标题2_translate.md  ← 第2个文件翻译结果
└── ...
```

### 3.5 工作流输出文件结构

运行 `webmdai workflow run` 后会生成：

```
E:\MyNovel\
├── TASK.md                      ← 你创建的任务文件（输入）
├── workflow.yaml                ← 你创建的工作流配置（输入）
├── TASK\                        ← 爬取结果（中间文件）
│   ├── TASK_1_xxx.md
│   ├── TASK_2_xxx.md
│   └── ...
├── TASK_cleaned\                ← 清理后的文件（中间文件）
│   └── ...
├── llm_translated\              ← LLM处理结果（中间文件）
│   └── ...
└── 全书合并版.md                 ← 最终成果（输出）
```

## 五、第二步：爬取网页

### 5.1 方式一：直接输入URL（适合少量）

```bash
# 基础格式
webmdai fetch batch \
  -w "网址1,网址2,网址3" \
  -n "任务名"

# 示例
webmdai fetch batch \
  -w "https://example.com,https://example2.com" \
  -n "我的任务" \
  -st
```

**参数说明**：

- `-w, --websites`: 网址列表，用逗号分隔
- `-n, --name`: 任务名称（决定输出文件夹名）
- `-s, --separate`: 每个网页单独保存
- `-t, --together`: 所有网页合并保存
- `-st`: 同时生成单独文件和合并文件（推荐）

### 5.2 方式二：任务文件（适合批量）

**创建任务文件 TASK.md**：

```markdown
# 任务

## 需要爬取的链接

- [第一个](https://nya.epheia.moe/example1)
- [第二个](https://nya.epheia.moe/example2)
- [第三个](https://nya.epheia.moe/example3)
- [第四个](https://nya.epheia.moe/example4)

## 说明
这是一些测试链接
```

**运行**：

```bash
webmdai fetch from-task TASK.md -st
```

**参数说明**：

- `TASKFILE`: 任务文件路径（默认 TASK.md）
- `-n, --name`: 覆盖任务名称
- `-r, --reader`: 爬取服务商（jina/firecrawl/direct）
- `-s/-t/-st`: 输出模式

### 5.3 交互模式（适合临时使用）

```bash
webmdai fetch interactive
```

然后按提示输入：

1. 输入网址（每行一个）
2. 输入 `/start` 结束
3. 输入任务名称

## 六、第三步：LLM处理

### 6.1 批量翻译

```bash
webmdai llm batch \
  -d "./我的任务" \
  -t translate \
  --separate \
  -o "./翻译结果"
```

**参数说明**：

- `-d, --directory`: 工作目录（包含要处理的文件）
- `-t, --task`: 任务类型（translate/summarize/explain/abstract）
- `-m, --model`: 模型别名（不指定用默认）
- `--separate`: 分别处理每个文件
- `--all`: 合并所有文件一起处理
- `-o, --output`: 输出目录

### 6.2 预设任务类型

| 任务          | 作用     |
| ----------- | ------ |
| `translate` | 翻译为中文  |
| `summarize` | 生成摘要   |
| `explain`   | 解释技术内容 |
| `abstract`  | 提取要点   |

### 6.3 自定义提示词

```bash
webmdai llm batch \
  -d "./文章" \
  -p "请将以下内容润色，使其更流畅：{content}" \
  --separate
```

注意：`{content}` 会被替换为文件内容

## 七、第四步：文本处理

### 7.1 批量替换文字

```bash
# 将"人工智能"替换为"AI"
webmdai deal batch \
  -d "./文章" \
  --text \
  -f "人工智能" \
  -r "AI"
```

**参数说明**：

- `-d, --directory`: 工作目录
- `--text`: 普通文本模式
- `--re`: 正则表达式模式
- `-f, --find`: 查找内容
- `-r, --replace`: 替换内容（留空=删除）
- `--preview`: 仅预览，不实际执行
- `--no-git`: 禁用Git备份

### 7.2 正则表达式替换

```bash
# 删除所有数字
webmdai deal batch \
  -d "./文章" \
  --re \
  -f "\d+" \
  -r "" \
  --preview
```

### 7.3 预览模式（强烈推荐先用）

```bash
# 加 --preview 先看看效果
webmdai deal batch \
  -d "./文章" \
  --text \
  -f "需要删除的广告文字" \
  --preview
```

## 八、工作流自动化（推荐）

工作流可以一次性完成：爬取→清理→翻译→替换→合并

### 8.1 创建工作流配置 workflow.yaml

```yaml
name: 轻小说翻译
description: 爬取、清理、翻译、合并的完整流程
version: "1.0"

variables:
  novel_name: "我的轻小说"
  author: "作者名"

stages:
  # 阶段1: 爬取
  - name: 爬取章节
    type: fetch
    enabled: true
    on_error: stop
    params:
      source: taskfile
      taskfile: TASK.md
      reader: jina
      delay: 1.5

  # 阶段2: 合并（简化版，跳过清理和LLM）
  - name: 合并全书
    type: merge
    enabled: true
    on_error: stop
    params:
      file_pattern: "*.md"
      sort_by: numeric
      output: "${novel_name}_全书.md"
      header: |
        # ${novel_name}
        ## ${author}
      add_chapter_headers: true
```

### 8.2 运行工作流

```bash
webmdai workflow run workflow.yaml
```

### 8.3 工作流阶段类型

| 类型        | 作用    | 主要参数                                      |
| --------- | ----- | ----------------------------------------- |
| `fetch`   | 爬取网页  | `source`, `taskfile`, `reader`, `delay`   |
| `clean`   | 清理内容  | `file_pattern`, `preset`, `output_suffix` |
| `llm`     | LLM处理 | `model`, `prompt_file`, `file_pattern`    |
| `replace` | 文本替换  | `file_pattern`, `replacements`            |
| `merge`   | 合并文件  | `file_pattern`, `output`, `sort_by`       |
| `command` | 执行命令  | `command`, `timeout`                      |

### 8.4 运行时可以设置变量

```bash
webmdai workflow run -v novel_name="另一部小说" -v author="另一个作者"
```

## 十、常见问题

### Q1: 爬取失败怎么办？

**可能原因**：

1. 网址无法访问
2. 被网站反爬
3. Jina Reader限流

**解决方法**：

```bash
# 1. 更换reader
twebmdai fetch from-task -r direct  # 使用直接爬取

# 2. 增加延迟
webmdai fetch from-task -r jina --delay 2.0

# 3. 检查网址是否能在浏览器打开
```

### Q2: LLM处理报错？

**可能原因**：

1. API密钥错误
2. 余额不足
3. 模型名称错误

**解决方法**：

```bash
# 检查模型配置
webmdai model list

# 确认端点格式正确（需要 /v1 结尾）
--endpoint https://openrouter.ai/api/v1
```

### Q3: 输出文件在哪？

**答案**：你在哪个目录运行命令，文件就在哪个目录生成。

```bash
# 假设你在 E:\Novels\MyBook 运行
cd E:\Novels\MyBook
webmdai fetch from-task
# 输出就在 E:\Novels\MyBook\TASK\ 下
```

### Q4: 怎么处理大量文件？

**建议**：使用工作流！

1. 创建 TASK.md 记录所有链接
2. 创建 workflow.yaml 配置处理流程
3. 一键运行 `webmdai workflow run`
4. 去喝杯咖啡，回来收成果
